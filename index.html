Introdução

Introdução

A Robótica está presente em praticamente todos os setores da indústria. Desenvolver robôs com sistemas confiáveis, que propocionem uma eficiência operacional na execução das suas funções e acima de tudo que tragam segurança aos seus utilizadores, estes são alguns dos pontos essenciais para os diferentes projetos do setor industrial. Um dos pilares da Indústria 4.0 é o desenvolvimento de robôs autônomos com um comportamento inteligente e que sejam cooperativos entre si. A Indústria 4.0 centra-se no desenvolvimento de processos mais autônomos e eficientes bem como oferecer soluções personalizadas para produção e logística. Para que isso seja possível utilizam-se tecnologias de automação industrial juntamente com sensores. O objetivo é criar um sistema produtivo mais inteligente e ampliar a capacidade de resolução de problemas sem a necessidade de interferência humana. No entanto, para que isso aconteça, é essencial haver uma constante troca de informações entre todas as etapas da cadeia de produção.

Toolbox Peter Corke

A disciplina de Sistemas de Robótica tem como principal objectivo a utilização do Matlab em conjunto com o Robotics Toolbox desenvolvida pelo Peter Corke, para construir e executar uma análise detalhada dos modelos integrados na toolbox.
O Robotics Toolbox é composto por um conjunto de ferramentas para simulação de robôs manipuladores. Conta com vários modelos de robôs comerciais pré-programados disponíveis para download no github ou no site: https://www.petercorke.com.

MDL_PLANAR2

O MDL_PLANAR2 é um script que cria a variável p2 no espaço de trabalho e descreve as características cinéticas de um simples mecanismo de ligação.
Este robot tem 2 graus de liberdade que se move tanto no eixo do XX como no eixo dos YY.







Matlab code

a1 = 1; a2 = 0.7; Rob = SerialLink([ Revolute('d', 0, 'a', a1, 'alpha', 0, 'standard') Revolute('d', 0, 'a', a2, 'alpha', 0, 'standard') ], ... 'name', 'planar 2 link'); qz = [0 0]; t=0:0.1:2; q1_pick=[pi/2 -pi/2]; q2_pick=[-pi/6 pi/4]; s1_pick = jtraj(q1_pick, q2_pick,t); plot (Rob, s1_pick, 'trail',{'r','LineWidth', 2}) q3_pick = [0, -pi/2] s2_pick = jtraj(q2_pick, q3_pick,t); plot(Rob, s2_pick) s3_return = jtraj(q3_pick, q1_pick,t) plot(Rob, s3_return) P = zeros (0,0) input ('Calculate the Position') q = Rob.getpos(); P(1) = a1*cos(q(1)) + a2*cos(q(1)+q(2)); P(2) = a1*sin(q(1)) + a2*sin(q(1)+q(2)); disp(P)

Revolute

Uma articulação de revolute é um par cinemático com um grau de liberdade. A articulação limita o movimento de dois corpos a uma rotação pura ao longo de um eixo comum. A junta não permite o movimento de translação nem o movimento linear deslizante. Quase todos os conjuntos de múltiplos corpos em movimento incluem as articulações revolute na sua construção. As juntas Revolute são utilizadas em numerosas aplicações tais como, dobradiças de porta, mecanismos e outros dispositivos de rotação de um único eixo. Neste trabalho utilizamos duas juntas revolute que correspondem aos dois graus de liberdade.





SerialLink

O SerialLink pode ser descrito como um conjunto de corpos rígidos (LINKS) numa corrente ligados por juntas. Cada articulação tem um grau de liberdade que pode ser de translação (articulação prismática) ou Rotacional (articulação revoluta).
Na toolbox RVC podemos ver o SerialLink, descrito como uma cadeia contendo "R" e "P". Esta notação representa o tipo de cada articulação da primeira à última. Por exemplo, "RRR" representa uma ligação em série com três juntas revolute.
A função SerialLink cria o robô utilizando os dados de cada uma das juntas, no caso deste trabalho temos que o robot é formado da seguinte forma:

Rob = SerialLink([ Revolute('d', 0, 'a', a1, 'alpha', 0, 'standard') Revolute('d', 0, 'a', a2, 'alpha', 0, 'standard') ], ... 'name', 'planar 2 link');

FANUC AM120iB/10L

O Modelo Fanuc AM120iB/10L é um robô com 6 graus de liberdade que usa os parâmetros Denvait-Hartenberg.
O conjunto de poses 6-D são obtidas, dada uma gama fixa de parâmetros alcançáveis, utilizando várias técnicas de cinemática corporal rígida e dinâmica.

Matlab code

Códifo Modelo Fanuc AM120iB/10L

clear all L(1) = Link([0 0 0.15 -pi/2 0]); L(2) = Link([0 0 0.77 pi 0]); L(3) = Link([0 0 0.1 -pi/2 0]); L(4) = Link([0 -0.96 0 pi/2 0]); L(5) = Link([0 -0.7 0 -pi/2 0]); L(6) = Link([0 0 0 0 0]); %########################################################## %Pose 0; At MASTERING position; %########################################################## Rob=SerialLink(L, 'name', 'Fanuc AM120iB/10L'); qf = ([pi/2 0 pi/4 pi/2 pi -pi/2]); Tf =Rob.fkine(qf); q0 =[-pi/2 0 pi/4 pi/4 pi -pi/2]; q =Rob.ikine(Tf, q0, 'mask',[1 1 1 1 1 1]); t = 0:0.15:8; Q= jtraj(q0, qf, t); Tr = fkine(Rob,Q); for i = 1:1:length(t) T = Tr(i); trs = transl(T); xx (i)=trs(1); yy (i)=trs(2); zz (i)=trs(3); end plot3 (xx, yy, zz, 'Color',[1 0 0], 'LineWidth', 2); hold on plot (Rob,Q);

Fanuc AM120iB/10L

O script do Fanuc AM120iB/10 disponível na Robotics Toolbox usa os paramêtros Denavit-Hartenberg para criar um modelo de robô. Foi definido o vetor de coordenadas conjunta q0 correspondente à posição inicial.
O Seriallink criará um robô definido pelos vetores de L(1) até L(6), correspondentes a cada uma das juntas.

Rob=SerialLink(L, 'name', 'Fanuc AM120iB/10L');

No decurso do estudo deste robô foi utilizada a cinemática direta e inversa.





Foward Kinematics - é usado para manipular a cinemática inversa, que posteriormente retornará a matriz de transformação homogênea final. A cinemática direta permite determinar a posição e a orientação do end-effector em função das variáveis das juntas do robô (x,y,z). É possível realizar essa análise fixando um sistema de coordenadas em cada elo. Para fazer isso de forma sistemática, foram utilizados os parâmetros de Denavit-Hartenberg (DH), que consistem num conjunto de quatro elementos que descrevem a posição e orientação de um sistema de coordenadas de um elo, em relação ao sistema de coordenadas do elo precedente ao longo da cadeia cinemática.
Os parâmetros de DH standard são definidos como:

i – Distância entre zi-1 e zi ao longo de xi;
αi – Ângulo entre zi-1 e zi ao redor de xi;
di – Distância entre xi-1 e xi ao longo de zi-1;
θi – Ângulo entre xi-1 e xi ao redor de zi-1.


Resolvendo a Cinemática Direta
qf = ([pi/2 0 pi/4 -pi/2 pi -pi/2]); Tf =Rob.fkine(qf)



Ikine - Cinemática Inversa

A cinemática inversa representa a transformação inversa da cinemática directa. A cinemática inversa possibilita determinar as variáveis das juntas em função da posição e orientação do end-effector. O conhecimento da solução do problema de cinemática inversa é indispensável para o controlo da trajetória ponto-a-ponto. Quando a posição do centro do end-effector não é alterada com o movimento das suas juntas, é possível utilizar o método de desacoplamento cinemático, que permite dividir o problema da cinemática inversa em dois problemas mais simples, conhecidos por cinemática inversa de posição (em função das juntas do braço), e cinemática inversa de orientação (em função das juntas do punho).
Resolvendo a Cinemática Inversa

q0 =[-pi/2 0 pi/4 pi/4 pi -pi/2]; q =Rob.ikine(Tf, q0, 'mask',[1 1 1 1 1 1]);
